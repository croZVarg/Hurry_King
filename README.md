# Project_NOSU
# 👑 Hurry King

## 1. Назначение проекта и правила игры
**Hurry King** — это пошаговая тактическая стратегия для двух игроков на одном компьютере. Цель игры — захватить как можно больше клеток игрового поля.

**Основные правила игры:**
* В начале игры на поле случайным образом генерируются стартовые замки игроков, препятствия (стены) и золотая шахта по центру.
* Игроки ходят по очереди. Новый замок можно построить в радиусе до 4 ячеек от своих текущих владений.
* **Механика задержки:** При постройке замка ячейки вокруг него сразу становятся «зоной влияния» (отображаются бледным цветом). Они превращаются в постоянные замки только в начале следующего хода этого же игрока.
* **Механика перехвата:** Враг может построить свой замок на вашей клетке влияния. Если вы строите замок на клетке влияния врага, 2 соседние клетки его влияния автоматически перекрашиваются в ваши.
* **Бонус золота:** Если зона влияния замка задевает золотую шахту, игрок получает право на бонусный ход.

## 2. Функциональные возможности
* **Кастомизация поля:** Возможность динамически задавать количество строк и столбцов перед стартом.
* **Динамическое масштабирование:** Автоматическая подгонка размеров ячеек под размер окна при его изменении.
* **Сохранение и загрузка:** Ручное сохранение текущего состояния сессии в текстовый файл и продолжение игры со стартового экрана.
* **Автоматический подсчет баллов:** Живой пересчет очков (количества замков) каждого игрока после каждого хода.
* **Эргономичный интерфейс:** Скрывающееся верхнее меню управления (`MenuStrip`), которое появляется только при наведении курсора.

## 3. Руководство администратора
Для успешного запуска и работы проекта необходимо:
* **Операционная система:** Windows 10 / Windows 11.
* **Среда выполнения:** .NET Framework версии 4.7.2 или выше.
* **Среда разработки (для сборки):** Visual Studio 2019 / 2022 (с установленной рабочей нагрузкой "Разработка классических приложений .NET").
* **Файловая структура:** В папке с запущенным `.exe` файлом должна находиться папка `images` с текстурами `bluecastle.jpg`, `redcastle.jpg`, `blueflag.jpg`, `redflag.jpg` (при их отсутствии игра автоматически переключается на цветную заливку ячеек).

## 4. Руководство пользователя
1. Запустите файл `HurryKingProbe.exe`.
2. На стартовом экране введите желаемое количество строк и столбцов в поля ввода.
3. Нажмите кнопку **«Начать игру»** (либо кнопку **«Продолжить игру»**, если у вас есть ранее сохраненная сессия).
4. В открывшемся игровом поле кликайте левой кнопкой мыши по доступным ячейкам для постройки замков и подтверждайте действие в диалоговом окне.
5. Чтобы сохранить игру, подведите курсор к самому верхнему краю окна — появится панель меню, где нужно нажать **Меню -> Сохранить игру**.

## 5. Руководство программиста (Структура проекта)
Проект реализован на архитектуре Windows Forms с разделением интерфейса и данных:
* **`StartScreen` (Form):** Стартовое окно приложения. Отвечает за ввод параметров игрового поля, валидацию данных и инициализацию новой или сохраненной игровой сессии.
* **`GameField` (Form):** Основное игровое окно. Содержит элемент `DataGridView` (`BattleField`), отвечает за визуальное отображение матрицы поля, обработку кликов по ячейкам, анимацию выезжающего меню `MenuStrip` и вызов методов игровой логики.
* **`CellState` (Enum):** Перечисление всех возможных состояний ячейки поля (`Empty`, `FirstCastle`, `SecondCastle`, `FirstOwned`, `FirstInfluence`, `SecondOwned`, `SecondInfluence`, `Obstacle`, `Gold`).
